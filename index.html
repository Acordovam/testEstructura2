Ejemplos de Git con GitHub

¿Qué es Git?

Git es un sistema de control de versiones distribuido que permite a los desarrolladores rastrear cambios en el código, colaborar con otros y mantener un historial detallado del proyecto.

¿Qué es GitHub?

GitHub es una plataforma de alojamiento de código basada en la web que utiliza Git. Facilita la colaboración entre desarrolladores, proporcionando herramientas para la revisión de código, la gestión de proyectos y la integración continua.

Clonar un repositorio

git clone https://github.com/usuario/repositorio.git

Este comando clona un repositorio existente desde GitHub a tu máquina local.

Proceso:

  1. Se copia todo el historial del repositorio.
  2. Se crea una nueva carpeta con el contenido del repositorio.
  3. Te permite trabajar en el proyecto localmente.

Hacer un commit

git commit -m "Mensaje del commit"

Este comando guarda los cambios en el repositorio local con un mensaje descriptivo.

Proceso:

  1. Los cambios en el Staging Area se agrupan en un commit.
  2. El commit se registra en el repositorio local con un identificador único.
  3. Se añade un mensaje que describe los cambios realizados.

Enviar cambios a GitHub

git push origin main

Este comando envía los commits del repositorio local al repositorio remoto en GitHub.

Proceso:

  1. Los commits locales se transfieren al servidor remoto.
  2. El repositorio remoto se actualiza con los últimos cambios.
  3. Otros colaboradores pueden ver y trabajar con los cambios enviados.

Actualizar repositorio local

git pull origin main

Este comando actualiza tu repositorio local con los cambios más recientes del repositorio remoto.

Proceso:

  1. Se descargan los cambios más recientes del servidor remoto.
  2. Se fusionan esos cambios con el repositorio local.
  3. Tu repositorio local queda sincronizado con el remoto.

Crear una nueva rama

git branch nombre-rama

Este comando crea una nueva rama en el repositorio local.

Proceso:

  1. Se crea un nuevo puntero que se puede mover independientemente del main.
  2. Permite trabajar en nuevas funcionalidades sin afectar la rama principal.
  3. Es útil para desarrollar características de manera aislada.

Fusionar ramas

git merge nombre-rama

Este comando fusiona la rama especificada con la rama actual.

Proceso:

  1. Los cambios de la rama especificada se integran en la rama actual.
  2. Se resuelven posibles conflictos que puedan surgir.
  3. La rama actual se actualiza con los cambios fusionados.
